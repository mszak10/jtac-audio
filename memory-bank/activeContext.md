## What was just built

- **Project scaffold**: Vite 5, React 18, TypeScript 5, Tailwind v3, `@crxjs/vite-plugin` v2 beta, Vitest 2.
- **Manifest V3** declared in `manifest.config.ts` with permissions `tabCapture`, `offscreen`, `activeTab`, `tabs`. Offscreen document registered as a web-accessible resource.
- **Shared logic** (`src/shared/`): `types.ts`, `messages.ts` (typed discriminated union + type guard), `routing.ts` (gain math, state map). All covered by Vitest.
- **Service worker** (`src/background/background.ts`): owns `Map<tabId, RoutingState>`, ensures the offscreen document exists, calls `chrome.tabCapture.getMediaStreamId`, forwards routing intent to offscreen, classifies capture errors, and clears state on tab close/reload.
- **Offscreen document** (`src/offscreen/offscreen.{html,ts}`): hosts a single `AudioContext`, builds a per-tab graph (`MediaStreamSource → ChannelSplitter → GainNode×2 → ChannelMerger → destination`), tears it down cleanly on stereo/close.
- **Popup** (`src/popup/`): minimal Tailwind UI with a three-way segmented control (Left / Stereo / Right). Reads current state from background on open, sends `SET_ROUTING` on click, surfaces capture errors with human copy.

## Notable deviation from `techContext.md`

- **shadcn/ui not installed.** The popup is small enough that hand-written Tailwind buttons are simpler than pulling in shadcn primitives. Add shadcn only when a real component (Dialog, DropdownMenu) is needed.
- **`@crxjs/vite-plugin` is on `^2.0.0-beta.34`.** The handbook calls for stable versions; v2 has no stable release yet and v1 doesn't support MV3 properly. This is a pragmatic exception. Pin and revisit when v2 stabilizes.
- **No Framer Motion / lucide-react yet.** Not needed for the popup's current scope. Add when there's a UI element that genuinely warrants them.
- **Icons generated programmatically.** `scripts/generate-icons.mjs` (pure JS via `pngjs`) writes 16/48/128 PNGs into `public/`. Design: dark rounded square, green/orange split circle. Regenerate via `node scripts/generate-icons.mjs`. Replace with a designed asset before Chrome Web Store submission if desired.

## Open / to verify

- **Offscreen document URL form.** `chrome.offscreen.createDocument` expects a path relative to the extension root. We pass `'src/offscreen/offscreen.html'` because that is what `@crxjs/vite-plugin` emits (verified in `dist/`). If the plugin changes the output path, update `OFFSCREEN_PATH` in `background.ts`.
- **`navigator.mediaDevices.getUserMedia` constraint shape.** Using legacy `mandatory` constraints with `chromeMediaSource: 'tab'`. This is the documented way to consume a `streamId` from `chrome.tabCapture.getMediaStreamId`. Cast through `MediaStreamConstraints` to satisfy TS.
- **AudioContext sample rate.** Currently lets the browser pick. If routing introduces audible artifacts, try matching the captured stream's sample rate.
- **Popup live updates.** Right now the popup only reads state on open; it does not subscribe to background changes. Acceptable while the popup itself is the only mutator.

## Things to remember

- **Test by ear.** TypeScript and Vitest can't validate audio. The only proof is plugging in headphones.
- **DRM-protected tabs cannot be captured** — Netflix / Spotify Web / some HBO content. Surface clearly, never silently fail.
- **Service workers die.** State lives in memory and is rebuilt on demand; that's fine for MVP. If state ever needs to survive a worker restart mid-session, move it into the offscreen document or `chrome.storage.session`.
- **Stereo means teardown** — when a tab returns to stereo, we stop capture and disconnect the graph rather than passing through at gain=1.
